## 发散现象

- 经常发生
- 发散后难以收敛
  - 人们常常忘记前面的逻辑
  - 人们没必要为了保持记忆而费心
- 发散是好事，应该鼓励而不是回避
- 但是应该置于有效交流的框架下

## this vs git

- git用于软件开发，其结构是沿着时间轴正向前进的。但是思想的网络与时间无关，而是与逻辑有关。
- this不需要像git一样merge，它的目的是顺着发散的趋势梳理出思想的网络。所以网络关系应当保留。
